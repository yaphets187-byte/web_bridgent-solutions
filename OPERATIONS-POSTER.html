Branch Logic

Use dev for normal work. Make changes, test them, and push safely without touching the live site.

Use main only for release. Push to main only when you want the production website updated.

  • local dev -> your computer working branch
  • origin/dev -> the dev branch on GitHub
  • local main -> your local production branch
  • origin/main -> the production branch on GitHub

Daily Work Checklist

  1. Open the repo bridgent-solutions.
  2. Confirm current branch is dev.
  3. Make changes.
  4. Check the change quickly on localhost:8888.
  5. Review changed files.
  6. Commit changes.
  7. Push dev.
  8. Check Netlify branch preview.
  9. Verify desktop and mobile layout.

Before Release Checklist

  1. Confirm localhost:8888 looks correct.
  2. Confirm Netlify preview looks correct.
  3. Confirm no unfinished work remains on dev.
  4. Test navigation and key sections.
  5. Test form if contact section changed.
  6. Confirm favicon / branding if assets changed.
  7. Check live-impacting copy carefully.

Netlify Rules

  • Keep GitHub integration enabled.
  • Production branch must stay main.
  • Enable branch deploys for dev.
  • Push to dev -> preview branch deploy.
  • Push to main -> production deploy.
  • Forms are handled by Netlify Forms.

Forms Checklist

  1. Submit a real test form when form-related changes are made.
  2. Check Netlify Forms -> consultation.
  3. Check email inbox aslesterr@bridgent-solutions.com.
  4. Check spam/junk if first-time delivery does not appear.

Publish Release

  1. Switch to main.
  2. Pull latest main.
  3. Merge dev into main.
  4. Push main.
  5. Wait for Netlify production deploy.
  6. Check the live site on desktop and mobile.

Do Not

  • Do not do routine work on main.
  • Do not push incomplete experiments to production.
  • Do not discard unknown Git changes blindly.
  • Do not remove Microsoft 365 DNS records while editing website DNS.

If Something Looks Wrong

  1. Check current branch first.
  2. Check whether changes were committed.
  3. Check whether changes were pushed.
  4. Check Netlify deploy status.
  5. Only then check browser cache.

Simple Workflow

1. Work Edit on dev only.
2. Save Check localhost, then commit and push dev.
3. Review Review the Netlify preview deploy.
4. Release Merge to main, push, and verify live site.
